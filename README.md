## Flow выполнения работ
В репозитории используется подход gitflow. Существуют 2 долгоживущие ветки:
- [production](https://github.com/allitov/simple-computer/tree/production) - содержит в себе результат выполнения всех практических заданий.
- [develop](https://github.com/allitov/simple-computer/tree/develop) - содержит в себе результат работы по текущему заданию.

### Начало работ
Для выполнения практического задания необходимо создать feature-ветку от ветки develop 
в формате 'feature-краткое-описание-задания'. В ней можно делать что душе угодно.

### Завершение работ
По завершении выполнения практического задания нужно создать pull request feature-ветки в ветку develop. После этого:
1) Проверить, что проходят все CI джобы.
2) Дождаться ревью от коллег (лучше сообщить об этом в чате).
3) Получить аппрув и влить ветку в develop.

Как только все feature-ветки будут влиты, [develop](https://github.com/allitov/simple-computer/tree/develop) 
вливается в [production](https://github.com/allitov/simple-computer/tree/production).

## Структура проекта
Содержимое каталогов проекта следующее:
- [console](./console) - содержит исходный код разрабатываемой консоли;
- [include](./include) - заголовочные файлы всех разрабатываемых библиотек;
- [myBigChars](./myBigChars), [myReadKey](./myReadKey), [mySimpleComputer](./mySimpleComputer), 
[myTerm](./myTerm) – исходные коды соответствующих библиотек и тестов для них;
- [simpleassembler](./simpleassembler) – исходный код транслятора с языка Simple Assembler;
- [simplebasic](./simplebasic) – исходный код транслятора с языка Simple Basic.

## Соглашения по разработке
- исходный код размещается в нескольких файлах;
- в одном файле исходного кода размещается ТОЛЬКО одна функция или 
ТОЛЬКО глобальные переменные для одной библиотеки (консоли);
- название каждого файла имеет вид <prefix>_имя содержащейся в
нем функции.c, для глобальных переменных используется файл с
именем вида <prefix>_variables.c, где <prefix> — это сокращенное
название соответствующей библиотеки. Для исходных файлов консоли префикс в имени файла не указывается.
- заголовочные файлы библиотек содержат весь публичный интерфейс библиотеки 
(функции и доступные пользователю глобальные переменные библиотеки)
и должны быть защищены от множественного повторного включения в исходный код. 
- Все заголовочные файлы располагаются в каталоге include.

## Исследования по работе
### Представление данных
Все данные представлены в виде 15 двоичных разрядов в [дополнительном коде](https://ru.wikipedia.org/wiki/%D0%94%D0%BE%D0%BF%D0%BE%D0%BB%D0%BD%D0%B8%D1%82%D0%B5%D0%BB%D1%8C%D0%BD%D1%8B%D0%B9_%D0%BA%D0%BE%D0%B4)
от -16384 до 16383 (0x0 - минимальное значение, 0x7FFF - максимальное значение).
Полезный [калькулятор](https://planetcalc.ru/embed/?id=zUAxQOrNLnd8M07vFtu_aFsd77HIFpwkUBmDbJyLRIfIoCKSbG4KtMkKsG0vkYkzK705EeoGlBPbaLemI-GLHLUDQd3RtdQo) для перевода из десятичной системы в двоичную.

_Примечание: В дополнительном коде нет привычного понятия отрицательного числа. Знак минус никто не ставит.
Положительные числа от 0x0001 (1) до 0x3FFF (16383), отрицательные от 0x4000 (-16384) до 0x7FFF (-1).
Ноль стандартный 0x0000._

### Оперативная память
Память состоит из 128 ячеек с адресами от 0 до 127 (0x0000 - минимальное значение, 0x007F - максимальное значение).

Каждая ячейка хранит 15 двоичных разрядов (0x0000 - минимальное значение, 0x7FFF - максимальное значение).

### Процессор

#### Регистры
Аккумулятор - временное хранилище данных и результатов операций, пока они не попали в оперативку.
15 бит (0x0000 - минимальное значение, 0x7FFF - максимальное значение).

Счетчик команд - содержит адрес ячейки в оперативке, где лежит текущая операция.
15 бит (0x0000 - минимальное значение, 0x7FFF - максимальное значение). **(Нужно уточнить, нет столько ячеек памяти)**
_Примечание: именно он в ответе за то, как отличить положительное число от операции!_

Регистр флагов - сигнализирует об определенных событиях. 5 бит. **(Уточнить соответствие флага и бита)**
